# Production Deployment Guide

Follow these steps to deploy the Employee Daily Pay Automation system to the cloud using **Render** (for the backend and database) and **Vercel** (for the frontend dashboard).

Once complete, the system will run 24/7 in the cloud without requiring your laptop to be powered on.

---

## Step 1: Upload Your Code to GitHub

1. Create a free account on [GitHub](https://github.com/) (if you don't have one).
2. Create a new **Private** repository named `afi-pay-automation`.
3. Open a terminal (PowerShell) in your project folder (`c:\Users\naaad\Desktop\afi`) and run the following commands to initialize Git and upload your code:
   ```powershell
   git init
   git add .
   git commit -m "Initialize project for production deployment"
   git branch -M main
   git remote add origin https://github.com/YOUR_GITHUB_USERNAME/afi-pay-automation.git
   git push -u origin main
   ```
   *(Note: The database and `.env` credentials are automatically excluded by `.gitignore` so they won't leak on GitHub).*

---

## Step 2: Deploy Backend & Database on Render.com

Render will use the `render.yaml` configuration file we created in the repository root to automatically launch a PostgreSQL database and your FastAPI backend.

1. Go to [Render.com](https://render.com/) and sign in.
2. In the top-right corner, click **New +** and select **Blueprint**.
3. Link your GitHub account and select your `afi-pay-automation` repository.
4. Under the Blueprint Configuration page:
   * Render will automatically detect the database and backend services.
   * Provide the values for the following environment variables:
     * **`WHATSAPP_ACCESS_TOKEN`**: Paste your permanent Meta System User token.
     * **`WHATSAPP_PHONE_NUMBER_ID`**: Paste your phone number ID (e.g. `1139995969204170`).
     * **`WHATSAPP_BUSINESS_ACCOUNT_ID`**: Paste your business account ID.
     * **`BOSS_WHATSAPP_NUMBER`**: Set your boss's number in international format (e.g., `+918925245191`).
5. Click **Apply**.
6. Render will now provision your database and deploy your backend. Once the build is complete, copy the backend URL generated by Render (it looks like `https://afi-pay-backend.onrender.com`).

---

## Step 3: Deploy Frontend Dashboard on Vercel

1. Go to [Vercel.com](https://vercel.com/) and sign in.
2. Click **Add New** and select **Project**.
3. Select your `afi-pay-automation` repository.
4. Under the project settings:
   * **Root Directory**: Click "Edit" and select the **`frontend`** directory.
   * **Framework Preset**: Vercel will automatically detect `Vite`.
   * Open the **Environment Variables** section and add the following variable:
     * **Key**: `VITE_API_URL`
     * **Value**: Paste your Render backend URL (e.g., `https://afi-pay-backend.onrender.com`)
5. Click **Deploy**.
6. Once deployed, Vercel will give you a public dashboard URL (e.g., `https://afi-pay-dashboard.vercel.app`). You can now open this URL on any phone or PC to access the dashboard!

---

## Step 4: Update Google Apps Script URL

Now that your backend is hosted permanently in the cloud, you can update your Google Form trigger to point directly to it!

1. Open your **Google Apps Script editor** browser tab.
2. Change **line 39** in your `Code.gs` script to use your new Render backend URL:
   ```javascript
   const backendUrl = 'https://YOUR_RENDER_SERVICE_NAME.onrender.com/api/submit-daily';
   ```
3. Save the script (**`Ctrl + S`**).

**That's it!** The system is now completely hosted on the cloud. You can turn off your laptop, and form submissions will still generate PDFs, update the dashboard, and send payslips to your boss's WhatsApp 24/7!
